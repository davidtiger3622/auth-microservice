Auth Microservice

A standalone authentication API for user identity and access management

v1.0.0

Service Status

Service is running — GET /health

Tech Stack

Node.js Express PostgreSQL JWT bcrypt Nodemailer express-validator helmet express-rate-limit

Authentication Endpoints

POST
/api/auth/register
Register a new user. Requires email and password (min 8 chars, uppercase, lowercase, number).
POST
/api/auth/login
Login with email and password. Returns JWT access token and refresh token.
GET
/api/auth/me
Get the currently authenticated user's profile.
Bearer Token
POST
/api/auth/logout
Logout and invalidate the refresh token server-side.
Bearer Token
POST
/api/auth/refresh-token
Exchange a valid refresh token for a new access token.
POST
/api/auth/forgot-password
Request a password reset link sent to the provided email address.
POST
/api/auth/reset-password
Reset password using the token received via email. Token is single-use and expires in 1 hour.

Security

INFO
bcrypt password hashing
All passwords are hashed with bcrypt at cost factor 12 before storage.
INFO
JWT token expiry
Access tokens expire in 15 minutes. Refresh tokens expire in 7 days.
INFO
Rate limiting
100 requests per 15 minutes per IP across all endpoints.
INFO
HTTP security headers
All responses include security headers via helmet.
